logic.auth module¶

logic/auth.py

Moduł auth.py zawiera funkcje odpowiedzialne za: - rejestrację nowych użytkowników (register) - logowanie istniejących użytkowników (login)

Funkcje operują na sesji SQLAlchemy i wykorzystują mechanizm haszowania haseł.

logic.auth.login(session: sqlalchemy.orm.Session, username: str, password: str) → User | None[źródło]¶

Próbuje zalogować użytkownika na podstawie podanego loginu i hasła.

Parametry:

  • session – aktywna sesja SQLAlchemy

  • username – nazwa użytkownika

  • password – hasło w postaci jawnej

Zwraca:

obiekt User, jeśli dane są prawidłowe; w przeciwnym razie None

logic.auth.register(session: sqlalchemy.orm.Session, username: str, password: str) → User[źródło]¶

Rejestruje nowego użytkownika w bazie danych.

Parametry:

  • session – aktywna sesja SQLAlchemy

  • username – nazwa użytkownika (unikalna)

  • password – hasło w postaci jawnej

Zwraca:

obiekt User zapisany w bazie

Wyrzuca:

ValueError – gdy użytkownik o podanej nazwie już istnieje

logic.habit_service module¶

logic/habit_service.py

Moduł logiczny odpowiedzialny za pobieranie nawyków i ich dzienników (logów) z bazy danych dla wskazanego użytkownika.

logic.habit_service.get_habit_logs_by_user_id(user_id: int) → List[HabitLog][źródło]¶

Zwraca listę wpisów dziennika nawyków (HabitLog) dla danego użytkownika. Do zapytania dołączony jest model Habit, aby móc korzystać z relacji.

Parametry:

user_id – identyfikator użytkownika

Zwraca:

lista obiektów HabitLog

logic.habit_service.get_habits_by_user_id(user_id: int) → List[Habit][źródło]¶

Zwraca listę wszystkich nawyków (Habit) przypisanych do danego użytkownika.

Parametry:

user_id – identyfikator użytkownika

Zwraca:

lista obiektów Habit
